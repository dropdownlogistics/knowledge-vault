Document Type Taxonomy · SYS-020
OBS
Observation
Documented pattern, finding, or behavioral tendency observed during operations. Operator sign-off only — no council vote required unless promoted to STD.
34+ entries · Governance category
STD
Standard
Codified rule governing DDL operations, architecture, or output quality. Council vote (LOCK / REVISE / REJECT) required before ratification.
~12 entries · Governance category
PRO
Protocol
Repeatable procedure governing a specific workflow or interaction pattern. Step-by-step process, not an enforceable rule.
~8 entries · Operational category
SYS
System
Full system definition — architecture, schema, governance layer. Each SYS entry corresponds to one of the 44 DDL systems.
44 systems · Reference category
ADR
Architecture Decision Record
Immovable architecture decision. Council review required. Once ratified, changes require a new ADR with explicit supersession. Feeds dex_decisions.

Recommendation
Non-binding guidance. Not an enforceable rule. May be promoted to STD when pattern recurs across systems.
